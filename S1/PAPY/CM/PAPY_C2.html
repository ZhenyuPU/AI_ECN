Paradigme : La programmation fonctionnelle¶

Un petit peu de théorie de la programmation¶

La programmation imprative est le paradigme le plus rpendu. On dcoupe le programme en une squence d'instructions excuter. C'est une description de l'algo excuter pour rsoudre le problme.
Offre un bon controle de la performmance.
La programmation dclarative considre le programme par une description du problme. Le moteur du language met en oeuve l'algo permettant la rsolution. C'est un paradigme de programmation sans effet de bord c..d. sans tat interne permanent. Exemple : Latex, CSS.
Formel donc permet de prouver que les rsultats sont corrects.
La programmation fonctionnelle est une variante de programmation dclarative. La programmation est vue comme l'valuation de fonctions mathmatiques sans effet de bord.
Langages fonctionnels : Haskell, Scala, OCaml

Par exemple:

In [ ]:

  • La mthode .sort est imprative. Elle modifie l'input avec une srie d'instructions.
  • La fonction sorted suit un modle fonctionnel, sans effet de bord. Deux appels successifs ne changent pas le rsultat.
Un programme utile aura toujours des effets de bords, il interragit avec le monde extrieur.

Python n'est pas un langage fonctionnel. Cependant, les recommandations de la programmation fonctionnelle:

  • limiter les effets de bord et tats internes
  • limiter les mutables
  • programmation qui caractrise la solution plutot que la procdure

permettent un style modulaire et donc compsable.

En pratique cela facilite aussi le teste et les dbugs et certaines acclrations (jit, paralllisme, GPU).

Les bibliothques pandas et altair ont une approche trs fonctionnelle. numpy beaucoup moins comme on le verra.

Deux mots sur les décorateurs¶

Un dcorateur de fonction (d'ordre suprieur) est un outil qui permet de marquer une fonction afin d'en modifier le comportement. Il est marqu par le @dcorateur avant une fonction.

from time import time
def timer(func:"fonction")->"fonction":
    name=func.__name__
    
    def timed_func(*args):
        t1=time()
        arg_str=', '.join(repr(arg) for arg in args)
        resultat=func(*args)
        elapsed=time()-t1
        print(f"{name}({arg_str}) elapsed time [{elapsed:0.8f}s]")
        return resultat
    return timed_func

Les classes¶

On va suivre l'excellente prsentation du livre de Xavier Olive, "Programmation python avance" qui utilise les "Boids"

Un Boid (bird-oid) est une reprsentation simplifie propose par Craig Reynolds pour modliser le comportement d'oiseaux avec les regles suivantes :

  • de sparation pour prserver une distance minimum entre les individus
  • de cohsion pour que les boids volent en groupe
  • d'alignement pour qu'ils volent dans la meme direction

class Boid:

    taille = 300 # dimension du domaine des positions
    count = 0

    def __init__(self, position=None, vitesse=None) -> None:
        self.x = (
            position
            if position is not None
            else np.random.uniform(-Boid.taille, Boid.taille, 2)
        )
        self.dx = vitesse if vitesse is not None else np.random.uniform(-5, 5, 2)
        Boid.count +=1
        
    def __repr__(self) -> str:
        return f"Boid({self.x.round(2)}, {self.dx.round(2)}, parmi {Boid.count})"
    
    def vitesse(self)-> float:
        return np.linalg.norm(self.dx)
    def avance(self) -> "Boid":
        return Boid(self.x+self.dx, self.dx)
    
    def __del__(self): #on implmente un destructeur pour s'assurer que count est mis  jours correctement
        Boid.count-=1
        print('__del__ was called')

Décorer ses objets¶

A dfaut d'attributs privs, on peut imposer certains comportements:

  • @property dfinit un raccourcis qui s'utilise comme une variable. (utile de l'ustiliser avec _var_name pour "protger" une variable.
  • @var_name.setter pour pouvoir modifier la valeur de la variable

On dfini une proprit vitesse de la faon suivante

@property
    def vitesse(self) -> float:
        return np.linalg.norm(self.dx)

    @vitesse.setter
    def vitesse(self, value: float) -> None:
        self.dx = self.dx * value / self.vitesse

De mme il est parfois utile de dfinir une nouvelle faon d'initialiser une instance de notre classe. Le dcorateur @classmethod le permet:

@classmethod
    def polaire(cls, r, theta, v, direction):
        return cls(r*np.array([np.cos(theta),np.sin(theta)]),v*np.array([np.cos(direction),np.sin(direction)]))

Enfin, on peut souhaiter encapsuler une "fonction" dans une classe par exemple pour controler sa visibilit. On utilisera le dcorateur @staticmethod lors de la dclaration de la fonction.

@staticmethod
def scene(n:int) -> "list[Boid]":
    return [Boid() for _ in range(n)]

Héritage et composition¶

Une classe fille peut hriter des proprits (et mthodes) d'une classe mre. Ceci permet de factoriser efficacement le code.

Attention, dans bien des cas, une relation de composition est plus judicieuse. Dans ce cas, un des attribus de la classe est un objet d'une autre classe.

In [ ]:
# exemple d'hritage: 
class Personne:
    def __init__(self, nom, prenom, age):
        self.nom=nom
        self.prenom=prenom
        self.age=age
        
    def __repr__(self):
        return f"{self.nom} {self.prenom} ({self.age}ans)"
    
    def gen_ID(self):
        return (self.nom+self.prenom[0]).lower()
    
    def anniversaire(self):
        self.age+=1
        print("Joyeux anniversaire")
    
class Eleve(Personne): #La classe Eleve herite de toutes les proprietes de Personne
    def __init__(self, nom, prenom, age):
        super().__init__(nom, prenom, age)
        self.student_ID="S"+self.gen_ID()
        
    def __repr__(self):
        return super().__repr__()+f" {self.student_ID}"

class Bulletin:
    def __init__(self, date, notes :"ndarray"):
        self.date=date
        self.notes=notes
        
    def moyenne(self):
        return np.average(notes)
    
    
class Eleve(Personne): #La classe Eleve herite de toutes les proprietes de Personne
    def __init__(self, nom, prenom, age):
        super().__init__(nom, prenom, age)
        self.student_ID="S"+self.gen_ID()
        self.bulletins=[]
        
    def __repr__(self):
        return super().__repr__()+f" {self.student_ID}"
    
    def meilleure_note(self):
        return np.array([B.notes for B in self.bulletins]).max()

Quelques conseils de structuration¶

Bien comprendre le problème et le cahier des charges¶

  • analyser le problme
  • quels sont les objectifs
  • quel structure gnrale est ncessaire
    • excutable
    • script
    • API
  • quelles sont les classes/variables de haut niveau
  • choisir le bon environnement de travail

Ebauche¶

  • crire un petit code test n'est pas interdit
  • crer les grandes structures et fonctions en premier, quitte les laisser vide mais commentes
  • "simple is beautiful" -> zen of python

Dev / Test¶

  • si possible crire les tests / appels API en premier -> contraint le dev dans la bonne direction
  • test driven programming?
  • s'assurer du bon fonctionnement des briques lmentaires au fur et mesure
    Rappel : dvelopper c'est environs 90% de dbuggage
    => Il faut absolument crire du code lisible!!
